## GE-Coding Convention(Style Guide)

编码规范很重要，架构师维持编码规范的几种方式

1. 人为约定 (Alibaba Java 开发手册等等)
2. 工具：插件
   - 后端 idea 的 alibaba java coding guide 、checkstyle、
   - 前端的 Eslint 等

#### 1. Submission Standard

1. 提交规范 (隐私保护)
2. 同一的提交记录，比如关联需求文档
3. 忽略某些文件的提交，保护隐私，可用使用.gitignore
